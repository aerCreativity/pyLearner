Lesson 2: List Methods

In addition to the primitive things we can do with lists, we also have access to a number of list methods. These are like the python functions we mentioned earlier like sum, but they are called on the list rather than given it, like as follows:

[1, 2, 3, 4].append(5)

Intuitively, the append method adds the given value to the end of the list it was called on. Some other methods we employ are .count() which returns the amount of times a given value shows up in a list, .index() which returns the earlier occurrence of a given value, .pop() which removes the last item and returns it, .remove() which removes the first occurrence of a given value from a list, .reverse() which reverses the order of the list, and .sort() which sorts the list in ascending order.

All of these list methods must be called after the list object, and must also be preceded by the dot operator. They cannot be called by themselves like len(), they must be done to a list.

You may also have noticed that many of these list methods modify the actual content of the list. When we discussed strings earlier, we noted that strings are immutable, or impossible to change, and the only way to really modify them was to create copies of them. This isn't true for lists; any changes we make to lists will actually apply to them as expected.

Task

Write a function listSwap that takes a list items, sorts it, reverses it, pops and prints the last element, and then prints the number of times lowercase a is present in the list.

Sample Input 0

listSwap(['e', 'c', 'd', 'a', 'b'])

Sample Output 0

a
0
